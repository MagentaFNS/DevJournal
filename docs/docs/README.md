# Документация разработчика DevJournal

Этот файл содержит техническую информацию для разработчиков, работающих над проектом.

## 📋 Описание проекта

**DevJournal** — это веб-приложение для ведения командного дневника разработчика с аналитикой прогресса. 
Проект состоит из:
*   **Бэкенда** на FastAPI (Python).
*   **Фронтенда** (статический сайт).
*   **Базы данных** SQLite (по умолчанию).

## 🚀 Быстрый старт (для новых участников)

1.  **Клонируйте репозиторий:**
    ```bash
    git clone https://github.com/MagentaFNS/DevJournal.git
    cd DevJournal
    ```

2.  **Установите и активируйте виртуальное окружение:**
    ```bash
    # Windows
    python -m venv venv
    venv\Scripts\activate

    # macOS / Linux
    python -m venv venv
    source venv/bin/activate
    ```

3.  **Установите зависимости:**
    ```bash
    pip install -r backend/requirements.txt
    ```

4.  **Запустите сервер разработки:**
    ```bash
    cd backend
    uvicorn app.main:app --reload
    ```
    API будет доступно по адресу `http://127.0.0.1:8000`.

## 🏗️ Архитектура и структура проекта

*   **`/backend`** — код серверной части.
    *   `app/main.py` — точка входа в приложение.
    *   `app/models.py` — модели данных для SQLAlchemy.
    *   `app/routers/` — эндпоинты API (разделены по функционалу).
*   **`/frontend`** — статические файлы веб-интерфейса (HTML, CSS, JS).
*   **`/docs`** — техническая документация и руководства для разработчиков.

## 🔌 Основные API-эндпоинты

| Метод | Путь | Описание |
| :--- | :--- | :--- |
| `GET` | `/api/entries` | Получить все записи дневника |
| `POST` | `/api/entries` | Создать новую запись |
| `GET` | `/api/analytics` | Получить данные для графиков |

## 📝 Рабочий процесс (Git)

1.  **Перед началом работы** всегда получайте актуальные изменения: `git pull`.
2.  **Создавайте новые ветки** для каждой новой функции или исправления: `git checkout -b feature/название-задачи`.
3.  **Делайте маленькие, логически завершённые коммиты** с понятными сообщениями.
4.  **Отправляйте изменения в свой форк** и создавайте Pull Request в ветку `main`.
5.  **Следите за обсуждениями** в Pull Request'ах и отвечайте на комментарии.
